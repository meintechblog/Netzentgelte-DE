# README Design

## Ziel

Die GitHub-README soll zwei Dinge gleichzeitig leisten:

1. Das Projekt fuer Besucher sofort verstaendlich machen.
2. Fuer Maintainer und Mitwirkende als belastbarer Einstieg in Setup, Architektur und Betrieb funktionieren.

## Empfohlener Stil

Die README wird als Hybrid aufgebaut:

- oben produktorientiert
- unten betrieblich und technisch

Sie soll nicht wie eine Marketing-Landingpage klingen, aber auch nicht wie eine reine interne Befehlsliste.

## Pflichtinhalte

- klares Projektversprechen in 2-3 Saetzen
- Live-Links
- Kernfunktionen
- aktueller Projektstatus
- Architektur mit LXC + statischem Hetzner-Publish
- Setup- und Entwicklungsbefehle
- Deployment- und Qualitaetsmodell
- Verzeichnisuebersicht
- naechste Baustellen / Roadmap

## Wichtige Botschaften

- `Netzentgelte Deutschland` ist eine eigene Datenplattform fuer Netzbetreiber- und `§14a`-Informationen.
- Oeffentlich sichtbar werden nur belastbare, verifizierte Daten.
- Entwicklung und Datenarbeit laufen auf dem LXC.
- `kigenerated.de/netzentgelte/` ist ein statischer Read-Snapshot, nicht die eigentliche Datenverarbeitungsinstanz.
- Bestandskorrekturen sind Teil des normalen Workflows, nicht nur neue Betreiberimporte.

## Ton

- direkt
- technisch serioes
- keine Buzzwords
- keine unhaltbaren Versprechen
- Zahlen nur dort, wo sie aktuell belastbar aus dem Projektstand ableitbar sind
